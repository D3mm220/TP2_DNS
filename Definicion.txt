Domain Name System o mejor conocido como DNS, es un sistema de nomenclatura que se encarga de traducir los nombres que tipeamos para que sean traducidos en direcciones IP, para 
comunicarse dentro de la red.
	Su función es la siguiente: Si tratamos de conectarnos a una página desde nuestra computadora, lo primero que nuestro dispositivo va a hacer es fijarse si en la caché DNS local
ya se encuentra el dominio traducido (dirección IP) por si anteriormente el dispositivo se había conectado con esa dirección. Si no se encuentra la caché, el próximo paso es pedirle 
al resolver (servidor dns local) que tiene conexión con el dns del ISP (Internet Service Provider). El dispositivo le pide al resolver que le consiga la dirección IP de la página a 
la que quiere acceder y el resolver se fija en su caché. Si este no tiene tampoco la dirección, el resolver acude a la ayuda con uno de los servidores raíces (Root Server). El servidor
raíz es el inicio de la jerarquía de DNS, está compuesto por 13 conjunto de servidores raíces distribuidos por todo el mundo y operados por 12 organizaciones,y cada conjunto tiene 
su dirección IP única. El resolver le pide la dirección IP de la página y el Root Server le pasa la dirección del servidor TLD para que le pregunte a este, ya que el root server no
se encarga de guardar las direcciones IP de direcciones específicas.
	Los servidores TLD (Top Level Domain) son servidores que almacenan la información de direcciones de dominios de nivel superior (.com, .net, .org, etc), este tampoco guarda 
direcciones IP específicas. El resolver le pide la dirección IP y este como no le puede dar la deseada le pasa la dirección para conectarse con el Servidor de Nombres Autoritario. 
Este se encarga de saber todo acerca del nombre de los dominios.(Las diferentes url de un mismo dominio por ejemplo: youtube.com y music.youtube.com). El resolver le pide la 
dirección IP y el Servidor de Nombre Autoritario finalmente le da la dirección pedida. El resolver guarda la dirección en su caché DNS y le pasa la dirección al primer 
dispositivo, haciendo que este también actualice su caché DNS. Finalmente, el dispositivo realiza comunicación con la página deseada
